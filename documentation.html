Abstract

TeVi is a website created as the product of a university project for the Web Technologies course that is part of the FII curriculum for year 2. The initials stand for "Terrorism Viewer", and the website deals with using a broad database of terrorist attacks in order to display their geographical locations on a map, or perform analytics operations on them by organizing charts. This document describes how the website functions, how to use it, and other implementation details.

Technologies

The technologies used for this project are as follows: the html/css, javascript, PHP languages, and the Leaflet, Leaftlet EasyPrint and Google Charts javascript libraries, and the MongoDB database solution.

Html/css was used for the website's appearance, while Js and PHP handle most of the website functionality. PHP is responsible with communication between components, as well as with the database, while Js is responsible for running the libraries that provide the core website functions.

Leaflet is an open-source mapping library for javascript, and it is used to generate the terrorism viewer map, as well as populate it with tokens marking the attacks. Leaftlet EasyPrint is a pure-Js plugin that allows the exporting of the map in .png format.

Google Charts is another open-source library for javascript, utilized for generating the charts that display the data detailing the terrorist attacks.

MongoDB is the project's chosen database solution due to its combination of efficiency and ease-of-use, as well as the fact the team was already familiar with it.

Implementation Details

The main functionalities offered by the website are the terrorism charts (implemented by Sebastian Ciobanu) and the terrorism map (implemented by Codrin Robotă), both of which make use of the database connection (implemented by Daria Popovici). The project follows mvc architecture, the implementation of which was primarily overseen by Daria Popovici.

Both charts and map function through the usage of a form detailing what sorting options to apply, and is filled out by the user then sent to the server via GET functionality. GET is used as opposed to POST because the data being sent is not sensitive, and benefits from being turned into link parameters that can be remembered. The server then turns the sent form data into MongoDB query parameters, and fetches the requested entries, handing them to the script that will then deal with the generation of either the map or the charts, using json format. The scripts will then iterate through the json entries and generate the chart or map from them.

Additionaly, the map also has an export functionality that is handled by the EasyPrint library, and allows the user to download a .png version of the map currently rendered with all the markers on it.

Use Cases

The normal usage case is for a user to access the site (via the homepage, or with a direct link to a subpage), navigate to the map/charts page, fill out the form with the desired data and submit it. Then, the server will return the requested map/chart, populated with the relevant information; this might take some time if the volume requested is particularly large.

Another normal usage case is for the user to click the "View Full Map" button in the lower left part of the map to get a full-page view of the map. From the full page view, he can click the export button present in the upper left side of the map, which will prompt the browser to download the .png version of the map.

The most common abnormal use case would be that in which the form is filled with wrong data, in which case the database query will not find any matching data, and the resulting map or chart will be empty.

Conclusion

The website provides an adequate representation of the terrorist attack data that a user might be interested in, as the map and charts can be easily understood and navigated. Further development of this application would entail broadening the chart types and map types further, as well as possibly offering some analytics functionalities such as calculating the median of a value, the average, and so on.
